## 📦 1. Basic Data Structures (मूल डेटा स्ट्रक्चर)
डेटा को मेमोरी में स्टोर करने के तरीके:

| Topic | Explanation (आसान शब्दों में) | Level |
| :--- | :--- | :---: |
| **Array** | एक कतार में रखे हुए डिब्बे। इंडेक्स (index) से वैल्यू तुरंत (O(1)) मिल जाती है। | 🟢 Easy |
| **String** | अक्षरों (Characters) का समूह (जैसे `"hello"`)। यह भी एक तरह का Array ही है। | 🟢 Easy |
| **Hash Table** | Dictionary की तरह। एक Key (जैसे Name) दो और तुरंत Value (जैसे Phone Number) मिल जाएगी। (O(1) time). | 🟢 Easy |
| **Linked List** | डिब्बों की ट्रेन, जहाँ हर डिब्बा अपने अगले डिब्बे का पता (pointer) जानता है। | 🟡 Medium |
| **Doubly Linked List** | ट्रेन का डिब्बा जो अगले (next) और पिछले (prev) दोनों डिब्बों का पता जानता है। | 🟡 Medium |
| **Stack** | प्लेटों का ढेर (LIFO - Last In First Out)। जो प्लेट सबसे बाद में रखी, वो सबसे पहले उठेगी। | 🟢 Easy |
| **Queue** | टिकट की लाइन (FIFO - First In First Out)। जो पहले आया, उसे पहले टिकट मिलेगा। | 🟢 Easy |

## 🛠️ 2. Core Techniques & Algorithms (सॉल्व करने के तरीके)
एरे या लिस्ट के ऊपर इस्तेमाल होने वाले पैटर्न्स:

| Topic | Explanation (आसान शब्दों में) | Level |
| :--- | :--- | :---: |
| **Prefix Sum** | पहले से जोड़कर रखना। जैसे: `[1, 2, 3]` -> `[1, 3, 6]`. रेंज सम (Range Sum) निकालने में काम आता है। | 🟢 Easy |
| **Two Pointers** | एरे में दो इंडेक्स (pointers) का इस्तेमाल करना (जैसे एक शुरुआत में और एक अंत में)। | 🟡 Medium |
| **Sliding Window** | एक मूविंग फ्रेम (Window)। सब-एरे (Sub-array) की प्रॉब्लम्स के लिए बेस्ट। | 🟡 Medium |
| **Binary Search** | आधी-आधी रेंज को रिजेक्ट करना। (Sorted डेटा में ढूँढने का सबसे तेज़ तरीका O(log N)). | 🟢 Easy |
| **Bit Manipulation** | बाइनरी बिट्स (0 और 1) के साथ खेलना (AND, OR, XOR, SHIFT)। | 🟡 Medium |
| **Sorting** | डेटा को बढ़ते या घटते क्रम में लगाना। | 🟢 Easy |
| **Recursion** | एक फंक्शन जो खुद को ही कॉल करता है जब तक कि अंत न आ जाए। | 🟡 Medium |
| **Divide and Conquer** | बड़ी प्रॉब्लम को छोटे हिस्सों में बांटना, उन्हें सॉल्व करना और फिर जोड़ना (जैसे Merge Sort)। | 🟡 Medium |

## 🌲 3. Trees & Hierarchical Structures (पदानुक्रमित डेटा)
पदानुक्रम (Hierarchy) दिखाने वाले डेटा स्ट्रक्चर्स:

| Topic | Explanation (आसान शब्दों में) | Level |
| :--- | :--- | :---: |
| **Tree** | एक नोड (Parent) जिसके नीचे कई बच्चे (Children) हो सकते हैं। कोई लूप नहीं होता। | 🟡 Medium |
| **Binary Tree** | ऐसा पेड़ जहाँ हर नोड के अधिकतम **दो** बच्चे (left, right) हो सकते हैं। | 🟡 Medium |
| **Binary Search Tree (BST)** | स्पेशल बाइनरी ट्री। लेफ्ट बच्चा हमेशा छोटा और राईट बच्चा हमेशा बड़ा होता है। | 🟡 Medium |
| **Heap (Priority Queue)** | एक ट्री जिसमें सबसे बड़ी (या सबसे छोटी) वैल्यू हमेशा टॉप पर (रूट) होती है। | 🟡 Medium |
| **Trie** | शब्दों की डिक्शनरी। Auto-complete या स्पेल चेक करने के लिए इस्तेमाल होता है। | 🔴 Hard |
| **Segment Tree / Fenwick Tree** | RANGE QUERY (जैसे 2 से 10 के बीच का सम निकालना और अपडेट करना) को O(log N) में करने के लिए। | 🔴 Hard |

## 🕸️ 4. Graphs & Connectivity (नेटवर्क और कनेक्शन)
कनेक्टेड नोड्स का नेटवर्क:

| Topic | Explanation (आसान शब्दों में) | Level |
| :--- | :--- | :---: |
| **Graph Theory** | नोड्स (Vertices) और उनके बीच के रास्तों (Edges) का नेटवर्क। | 🟡 Medium |
| **DFS (Depth-First Search)** | एक रास्ते पर अंत तक जाना, फिर वापस आकर दूसरा रास्ता चुनना। | 🟡 Medium |
| **BFS (Breadth-First Search)** | आसपास के सारे रास्तों को एक साथ एक-एक कदम आगे बढ़ाना। | 🟡 Medium |
| **Union-Find (DSU)** | ग्रुप्स को आपस में जोड़ना और यह पता लगाना कि दो लोग एक ही ग्रुप में हैं या नहीं। | 🔴 Hard |
| **Topological Sort** | कामों को सही क्रम में लगाना। (जैसे: कोर्स B करने के लिए कोर्स A पहले करना जरूरी है)। | 🟡 Medium |
| **Shortest Path** | पॉइंट A से B तक का सबसे छोटा रास्ता निकालना (Dijkstra algorithm). | 🔴 Hard |
| **Minimum Spanning Tree (MST)** | सारे नोड्स को सबसे कम खर्च (minimum edge weight) में आपस में जोड़ना। | 🔴 Hard |

## 💡 5. Advanced Optimization & Logic (एडवांस्ड एप्रोच)
बड़े और कठिन प्रॉब्लम्स को हल करने की ऑप्टिमाइज्ड एप्रोच:

| Topic | Explanation (आसान शब्दों में) | Level |
| :--- | :--- | :---: |
| **Greedy** | जो अभी सबसे बेस्ट दिख रहा है, उसे चुन लो (बिना भविष्य की सोचे)। | 🟡 Medium |
| **Backtracking** | सारे पॉसिबल रास्ते आजमाना। अगर गलत रास्ता मिले, तो वापस कदम पीछे लेना (जैसे Sudoku). | 🟡 Medium |
| **Dynamic Programming (DP)** | रिपीट होने वाले सब-प्रॉब्लम्स के आंसर्स को स्टोर कर लेना ताकि बार-बार कैलकुलेट न करना पड़े। | 🔴 Hard |
| **Memoization** | DP का ही एक तरीका (Top-Down)। पिछले आंसर्स को एक टेबल/मैप में याद रखना। | 🟡 Medium |
| **Monotonic Stack/Queue** | ऐसा स्टैक/क्यू जो हमेशा Sorted (बढ़ते या घटते) क्रम में रहता है। | 🔴 Hard |
| **Bitmask** | 0 और 1 के कॉम्बिनेशन से स्टेट्स (states) को रिप्रेजेंट करना। | 🔴 Hard |

## 🔢 6. Mathematics & Specialized Sorting (गणित और विशेष सॉर्टिंग)
मैथमेटिकल टॉपिक्स और सॉर्टिंग एल्गोरिथम्स:

| Topic | Explanation (आसान शब्दों में) | Level |
| :--- | :--- | :---: |
| **Number Theory / Math** | प्राइम नंबर्स, GCD, LCM, और मॉड्युलर अरिथमेटिक के रूल्स। | 🟢 Easy |
| **Combinatorics** | परम्यूटेशन और कॉम्बिनेशन ($nCr, nPr$) - तरीके गिनना। | 🟡 Medium |
| **Geometry** | पॉइंट्स, लाइन्स, और आकृतियों के बीच की दूरी या एरिया निकालना। | 🟡 Medium |
| **Game Theory** | दो प्लेयर्स का खेल (जैसे Tic-Tac-Toe)। कौन जीतेगा या ड्रॉ होगा, यह प्रेडिक्ट करना। | 🔴 Hard |
| **Counting Sort / Radix / Bucket** | बिना कम्पेरिज़न किए O(N) टाइम में सॉर्ट करने के स्पेशल तरीके। | 🟡 Medium |
| **Quickselect** | बिना पूरे एरे को सॉर्ट किए $k$-th सबसे बड़ा एलिमेंट ढूँढना (O(N) time). | 🟡 Medium |

## ⚙️ 7. System Concepts & Utilities (यूटिलिटी और अन्य)
डिजाइन और विशेष टूल्स:

| Topic | Explanation (आसान शब्दों में) | Level |
| :--- | :--- | :---: |
| **Design** | कोई खास क्लास या डेटा स्ट्रक्चर खुद डिजाइन करना (जैसे LRU Cache design). | 🟡 Medium |
| **Data Stream** | लगातार बहता हुआ डेटा। जैसे live traffic data में से median निकालना। | 🟡 Medium |
| **Simulation** | जैसा सवाल में बोला गया है, कोड में बिल्कुल वैसा ही करके दिखाना (जैसे रोबोट का चलना)। | 🟢 Easy |
| **Interactive** | जिसमें आपको API कॉल करके या सिस्टम से पूछ-पूछ कर जवाब ढूँढना होता है। | 🟡 Medium |
| **Concurrency** | मल्टी-थ्रेडिंग। एक साथ कई काम बिना आपस में टकराए कैसे चलाएं। | 🔴 Hard |
| **Database / Shell** | SQL क्वेरीज़ या Bash/Linux कमांड्स लिखना। | 🟢 Easy |
